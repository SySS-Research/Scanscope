Info

Scanscope presents information from a port scan for humans.

The Charts

The charts can look confusing at first glance, so here is some clarification. Let's go over some important details.

  • Host group: All hosts that have the same set of ports open belong to the same host group. Filtered ports count as closed.
  • Fingerprint: For technical reasons, we assign a set of ports a fingerprint, which is a hexadecimal number with 32 digits. There is one special case: The emtpy set, i.e. an active host with no open ports, is assigned a NULL value.
  • Color: When choosing the hash-based color scheme, each fingerprint is assigned a color to create a visual aid for differentiating between host groups. It has no special meaning. Because there are many fingerprints, colors can be very similar. This is a natural consequence and a known limitation. The NULL fingerprint is assigned the color grey.
  • TCP/UDP ports: For brevity, we leave out the designation TCP/UDP. Instead, we define positive port numbers as TCP and negative port numbers as UDP; i.e. port -53 is 53/udp. The top 14 used ports (according to Nmap) are color coded.
  • Categories: Each host group is assigned a category by associating common ports with a service. Some hosts expose ports that could be assigned to different categories, so be aware that this classification is not unambigious.
  • Coordinates: The coordinates in the 2D bubble chart are derived by projecting points in a 217 dimensional space onto two dimensions using algorithms from data science. They have no particular meaning. The aim is that similar host groups are nearby.
  • Cluster: When using the HDBSCAN method, host groups are assigned to clusters based on the similarity of their port configurations. Hosts in the same cluster share similar characteristics and are positioned near each other in the visualization. Outliers that don't fit well into any cluster are marked separately.
  • Limitations: As usual, firewalls and other network components between your device and the targeted systems can skew the results of a port scan. Also, the more ports you scan the better the result. In terms of Nmap arguments, -p- (i.e. all TCP ports) is best while -F (i.e. the top 100 ports) should represent a reasonable balance between speed and meaningfulness of the results.
